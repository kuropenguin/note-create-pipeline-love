# 記事執筆スキル

`articles/<slug>/plan.md` の企画に基づき、note.com 向けの記事を執筆する。

## 手順

1. `$ARGUMENTS` から slug を取得する（例：`/write-article love-after-breakup`）。

2. `articles/<slug>/plan.md`（企画書）を読み込む。

3. 企画書に記載された元データの行を `data/reddit_master.csv` から読み込む。**記事本文とコメントを全て読むこと**（サマリーだけで書かない。元投稿の具体的なエピソード・セリフ・コメント欄の反応を把握した上で執筆する）。

4. 同ディレクトリの `guidelines.md` を参照し、**バズる記事の書き方ルール**に沿って記事を執筆する。

5. 同ディレクトリの `article-template.md` の構成に沿ってマークダウンで記事を書く。

6. `articles/<slug>/article.md` に出力する。

7. **画像（任意）**：画像が記事の価値を高めると判断した場合のみ作成する。画像なしの記事でも全く問題ない。
   - 画像を入れる場合は、記事本文中に `<!-- ここに画像を貼る: 01-header -->` のようなコメントを記述する
   - 対応する画像生成プロンプト（**日本語**）を `articles/<slug>/image-prompts/` に保存する
   - ファイル名は挿入箇所がわかる命名にする：`01-header.txt`, `02-body1.txt`, `03-body2.txt` 等
   - 各プロンプトには画像のスタイル、雰囲気、構図を具体的に記述する
   - **必須**：`guidelines.md` の「画像プロンプトのベーススタイル」に定義された共通トンマナを必ず適用すること（フィルム写真風・顔を写さない・共通フレーズ末尾付与など）

## 重要な執筆ルール

- **元ネタの Reddit 投稿を直訳しない**。エッセンスを抽出し、日本の読者向けにローカライズする
- **個人情報は完全に匿名化**する（名前、地名、年齢等は変更）
- **一人称視点**で書く（「私」or 三人称の体験談風）
- `guidelines.md` の AI 臭い表現リストを必ず避ける
- **有料記事の場合**：記事中に有料ラインを明示する（`article-template.md` の有料ライン指示に従う）
- **記事の最後**に自然な流れでスキ（いいね）をお願いする一文を入れる。短く、記事のトーンに合わせること
