# 記事作成パイプライン

素材選定 → 執筆 → AIレビューループ → 人間レビュー → 完了 の全フローを実行する。

## フロー

### ステップ0: 記事タイプの決定

`data/article_log.csv` を読み込み、過去の記事タイプの比率から今回のタイプを自動決定する。

**目標比率:**
- **無料記事**: 7/10
- **無料記事（有料紹介）**: 2/10 — 無料記事だが、既存の有料記事を本文中にさりげなく紹介する（「このXXXについてはこちらの記事で詳しく解説しています」程度）
- **有料記事**: 1/10

**判定ロジック:**
1. ログが空 or 有料記事が0件 → まず有料記事が1本必要なので「無料記事」を書く（有料紹介には紹介先がないため）
2. 有料記事が1件以上あり、直近10件に有料記事が0件 → **有料記事**
3. 有料記事が1件以上あり、直近10件に無料(有料紹介)が2件未満 → **無料記事（有料紹介）**
4. それ以外 → **無料記事**

決定した記事タイプを以降のステップに引き継ぐ。

### ステップ1: 素材選定

`/select-material $ARGUMENTS` を実行する。
- ユーザーがキーワードや行番号を指定した場合はそれを渡す
- 記事タイプ（無料/有料）を select-material に伝える
- 実行後、slug を取得する

### ステップ2: 記事執筆

`/write-article <slug>` を実行する。
- 記事タイプに応じて guidelines.md の無料記事編 or 有料記事編に従う
- **無料記事（有料紹介）** の場合：無料記事として書きつつ、`data/article_log.csv` から有料記事を1つ選び、本文中の関連する箇所でさりげなく紹介する
- `articles/<slug>/article.md` と画像プロンプトが生成される

### ステップ3: AIレビューループ（自動）

以下を**全レビュワーが PASS するまで**繰り返す：

1. `/review-article <slug>` を実行
2. `articles/<slug>/review.md` の総合判定を確認
3. **FAIL の場合**:
   - `/revise-article <slug>` を実行して修正
   - ステップ3の1に戻る（再レビュー）
4. **PASS の場合**:
   - ループ終了、ステップ4へ

**注意**: 無限ループ防止のため、最大3回までのリトライとする。3回 FAIL した場合は現状の記事と review.md をユーザーに提示し、判断を仰ぐ。

### ステップ4: 人間レビュー

AIレビューが全て PASS したら：

1. 以下を表示する：
   ```
   ✅ AIレビュー全パスしました。

   記事: articles/<slug>/article.md
   レビュー結果: articles/<slug>/review.md
   画像プロンプト: articles/<slug>/image-prompts/

   記事を確認してください。
   ```
2. ユーザーの返答を待つ（ここで停止する）

3. ユーザーの返答に応じて：
   - **「OK」「完了」「LGTM」等** → ステップ5（完了）へ
   - **フィードバックあり** → 以下を行う：
     1. `articles/<slug>/review.md` に人間レビューの内容を追記する（フォーマット下記）
     2. フィードバックに基づき記事を修正する
     3. ステップ3（AIレビューループ）に戻る

**人間レビュー記録フォーマット**（review.md に追記）：
```markdown
## 人間レビュー（YYYY-MM-DD）

### 指摘事項
- {ユーザーが指摘した内容を要約}
- {複数あれば箇条書きで}

### 対応
- {どう修正したかを簡潔に}
```

### ステップ5: 完了

1. `articles/<slug>/plan.md` と `articles/<slug>/article.md` から情報を読み取り、`data/article_log.csv` に記録する：
   ```bash
   echo '<JSON>' | python3 .claude/skills/article-pipeline/update_article_log.py
   ```
   JSON フォーマット:
   ```json
   {
     "slug": "<slug>",
     "タイトル": "記事の最終タイトル",
     "概要": "1行の記事概要",
     "コンセプト": "記事の切り口・テーマ",
     "バズり具合": "◎/○/△",
     "元ネタRedditタイトル": "reddit_master.csv の元タイトル",
     "記事タイプ": "無料/無料(有料紹介)/有料",
     "作成日": "YYYY-MM-DD"
   }
   ```
2. 最終版の記事情報を表示
3. commit & push する

## 各ステップの単独実行

各スキルは単独でも呼び出せる：
- `/select-material` — 素材選定のみ
- `/write-article <slug>` — 執筆のみ
- `/review-article <slug>` — レビューのみ
- `/revise-article <slug>` — 修正のみ
