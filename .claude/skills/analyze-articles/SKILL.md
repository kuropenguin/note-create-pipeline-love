# 記事分析スキル

`data/reddit_master.csv` の未分析行（サマリーが空の行）を分析し、サマリー・想定読者・タグを付与する。

## 手順

1. `data/reddit_master.csv` を Python で読み込み、サマリーが空の行のインデックスと内容を取得する。
   - 行数が多い場合は **バッチ処理**（10行ずつ）で進める。

2. 各行の「タイトル」「記事本文」「コメント」「ups」「upvote_ratio」「コメント数」を読み、以下を生成する。

### サマリー（以下のフォーマット厳守）

```
【元ネタ概要】
1行目：状況
2行目：何が起きたか／何が語られたか
3行目：コメント欄の反応や結末

【日米格差】小／中／大
【感情の具体性】高／中／低
【逆説・発見のタネ】あり／なし（あれば一言）
【記事化のしやすさ】◎／○／△

【候補タイトル】
① 具体的：（状況・人物・行動が明確なもの）
② 中間：（感情や構造を言語化したもの）
③ 抽象的：（普遍的なテーマに昇華したもの）
```

### 想定読者
- 記事が刺さりそうな読者層を1〜2行で記述する（例：「20〜30代の転職を考えている会社員」）

### タグ
- カンマ区切りで3〜5個のタグを付与する（例：「仕事,転職,人間関係,アメリカ文化」）

3. 生成結果を JSON 形式にまとめ、`update_csv.py` に渡して CSV を更新する。

```bash
echo '<JSON>' | python3 .claude/skills/analyze-articles/update_csv.py
```

JSON フォーマット:
```json
[
  {
    "row_index": 0,
    "サマリー": "...",
    "想定読者": "...",
    "タグ": "..."
  }
]
```

4. 更新後、処理件数と残件数を報告する。

## 注意事項
- 既存カラム（記事本文・コメント等）は一切変更しない
- サマリーが既に入っている行はスキップする
- CSV の文字コードは UTF-8 を維持する
- 大量行がある場合、ユーザーに確認してからバッチ数を決める
